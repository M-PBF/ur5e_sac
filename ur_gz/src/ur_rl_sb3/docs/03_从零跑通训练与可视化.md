## 1. 第一次运行（标准流程）


先跑 5000 步确认链路无误，再拉长步数。

## 2. 在 RViz 显示随机目标点

在 RViz 做这几步：

1. `Add` -> 选择 `Marker`
2. Topic 设为 `/ur_rl_sb3/goal_marker`
3. Fixed Frame 建议 `base_link`（或能看到机器人 TF 的全局 frame）

你会看到一个红色球体，每个 episode reset 后会更新位置。

## 3. 训练日志怎么看

现在训练脚本每回合结束会输出类似：

```text
[Episode 12] reason=max_steps, final_error=0.1842, ep_reward=-31.553, ep_len=200
```

字段解释：

- `reason=goal_reached`：成功达到目标
- `reason=max_steps`：没来得及达到，步数耗尽
- `final_error`：最终末端与目标点的距离误差
- `ep_reward`：整回合累计奖励
- `ep_len`：回合长度

观察趋势：

- `goal_reached` 次数逐步增加
- `final_error` 均值下降
- `ep_reward` 逐步提高

## 5. 新手最高频故障排查

## 5.1 改了 launch 但没生效

原因：只改了 `src`，未更新 `install`。

解决：

```bash
cd /home/bophy/d2lros2/ur5/ur_gz
colcon build --symlink-install
source install/setup.bash
```

## 5.2 Gazebo/RViz 不弹窗

排查：

```bash
echo $DISPLAY
```

- 空值通常代表当前会话无图形显示。
- 远程/容器环境需额外配置 X11。

## 5.3 仿真启动了但机械臂不动

检查：

```bash
ros2 topic echo /forward_position_controller/commands
```

- 如果没数据，训练脚本没在发命令。
- 如果有数据但不动，检查 controller 是否激活。

## 5.4 找不到包

检查：

```bash
ros2 pkg prefix ur_simulation_gz
ros2 pkg prefix ur_rl_sb3
```

查不到就说明当前终端环境没 source 好，或者包没 build。

## 5.5 Ctrl+C 后看到 gazebo exit code -2

这通常是中断退出导致的“非优雅退出”，多数场景不是功能错误。

## 6. 建议你的训练节奏（省时间）

- 第 1 轮：`--timesteps 5000` 验证流程
- 第 2 轮：`--timesteps 30000` 调动作尺度/阈值
- 第 3 轮：`--timesteps 150000` 出结果图

每轮都记录：

- 成功率
- 平均 final error
- 训练耗时

这三项就是你做项目汇报的核心指标。
