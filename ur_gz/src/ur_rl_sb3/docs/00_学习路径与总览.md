# UR5 + ROS2 + Gazebo + SB3 学习路径与总览（中文）

> 面向对象：完全零基础（ROS2 原理也不熟）
> 
> 目标：你能独立启动项目、训练策略、解释系统链路，并能应对实习面试。

---

## 1. 你现在这个项目到底在做什么？

一句话：

- **官方包 `ur_simulation_gz`** 负责搭建 UR5 仿真世界（机器人模型、Gazebo、控制器、话题桥接等）。
- **我们新增包 `ur_rl_sb3`** 负责把仿真封装成 RL 环境，并用 Stable-Baselines3 训练策略。

你可以把它理解为：

- 官方包 = “地基 + 管道 + 电路”
- 你的 RL 包 = “智能大脑”

## 3. 你要记住的 5 个“关键事实”

1. `ur_rl_sb3/launch/ur_rl_sim.launch.py` 不是重建世界，而是 `include` 官方 `ur_sim_control.launch.py`。
2. 只改 `src/` 不会立即生效，必须 `colcon build` + `source install/setup.bash`。
3. RL 环境核心是：订阅 `/joint_states`，发布 `/forward_position_controller/commands`。
4. RViz 目标点在 `/ur_rl_sb3/goal_marker`（`Marker`）。
5. 每回合结束都会打印原因：`goal_reached` 或 `max_steps`。

## 6. 常见误区（先避坑）

- 误区1：改了 launch 直接跑就生效 —— 错，需要 build + source。
- 误区2：Gazebo 开了就会自动训练 —— 错，训练脚本需要单独运行。
- 误区3：看到末端“乱动”就是成功 —— 错，要看误差和回合结束原因。
- 误区4：rviz 没看到目标点就是没发布 —— 先检查 Marker display 的 topic。
