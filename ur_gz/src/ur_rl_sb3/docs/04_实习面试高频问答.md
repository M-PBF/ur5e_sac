# 实习面试高频问答（针对本项目）

> 用法：先看“短回答版”，再看“展开版”，最后练“追问题”。

---

## Q1：你这个项目做了什么？

**短回答版**

我基于 ROS2 Jazzy 和 Gazebo 的官方 UR 仿真包，实现了一个 SB3 强化学习训练框架。核心是把仿真封装成 Gym 环境，定义状态、动作、奖励和终止条件，并加入 RViz 目标点可视化和每回合诊断日志。

**展开版**

- 复用官方仿真 launch（xacro、Gazebo、ros2_control、controller 管线）；
- 新增 `ur_rl_sb3` 包，训练算法用 SAC；
- 通过 topic/TF 获取状态并控制机械臂；
- 输出成功率、误差等指标用于评估。

---

## Q2：为什么不用自己从零搭仿真？

**短回答版**

工程上优先复用稳定底层，避免重复造轮子。我的重点是 RL 层和可解释训练链路。

**追问应对**

我没有回避底层，而是理解并复用了官方链路，再在其上做 RL 抽象，这更接近工业团队协作方式。

---

## Q3：你的 observation / action / reward 怎么设计？

**短回答版**

- Observation：关节位置、关节速度、末端位置、末端到目标误差；
- Action：6 维连续动作映射到关节增量命令；
- Reward：负距离 + 动作正则，成功额外奖励。

**追问：为什么加动作正则？**

为了抑制策略输出抖动和过激动作，提高控制稳定性和收敛质量。

---

## Q4：为什么选 SAC？

**短回答版**

任务是连续控制，SAC 在连续动作空间里通常更稳定、样本效率更好。

**可补充**

后续可与 PPO 做对比实验，形成实验结论。

---

## Q5：你怎么判断训练是否有效？

**短回答版**

看三类指标：

- `goal_reached` 回合比例（成功率）
- `final_error` 均值
- 回合奖励趋势

同时结合 RViz marker 可视化观察行为是否合理。

---

## Q6：你遇到过哪些坑？怎么解决？

**推荐回答（贴近真实）**

1. 改了 launch 不生效：
   - 原因：只改 `src`，忘了 `build + source`。
2. GUI 不出现：
   - 原因：图形环境/参数问题，或运行了 install 的旧 launch。
3. 机械臂不动：
   - 原因：训练脚本没发控制、控制器未激活、话题不匹配。
