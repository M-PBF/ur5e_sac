# ROS2 零基础入门（结合本项目）

> 本文目标：你不需要先学完 ROS2 全家桶，先掌握“跑项目必须的 20% 核心”。

---

## 1. ROS2 是什么？

可以把 ROS2 看成“机器人系统的中间件 + 工程规范”。

它解决的问题：

- 各模块（感知、控制、规划、仿真）如何通信？
- 如何启动一堆进程？
- 如何统一日志、参数、调试接口？

在这个项目里你会接触到：

- Node（节点）
- Topic（话题）
- Launch（启动编排）
- Workspace（工作区）
- Build/Install/Source（构建与环境加载）

---

## 2. ROS2 的 5 个核心概念

## 2.1 Node（节点）

节点就是一个运行中的功能进程。

例子：

- `robot_state_publisher`：发布机器人 TF
- `controller_manager`：管理控制器
- 你写的 RL 环境内部也创建了一个 node（`ur_rl_sb3_env`）

## 2.2 Topic（话题）

话题是发布/订阅通信总线。

- 发布者一直发消息
- 订阅者一直收消息

本项目常见话题：

- `/joint_states`：关节状态
- `/forward_position_controller/commands`：关节位置命令
- `/ur_rl_sb3/goal_marker`：RViz 目标点

## 2.3 Launch

launch 文件用于“按顺序拉起一组节点”。

你当前的 `ur_rl_sim.launch.py` 通过 include 官方 launch，把一整套系统拉起来。

## 2.4 Workspace

工作区是 ROS2 工程目录，一般结构：

- `src/`：源码
- `build/`：构建中间文件
- `install/`：安装产物（运行时主要用这里）
- `log/`：构建日志

## 2.5 Build + Source

这是新手最容易混淆的：

- `colcon build`：把 `src` 编译/安装到 `install`
- `source install/setup.bash`：让终端“认识”刚刚 build 出来的包

只改 `src` 不 build/source，运行时可能还是旧代码。

## 3. 你会常用的命令（必须会）

## 3.1 查包

```bash
ros2 pkg prefix ur_simulation_gz
ros2 pkg prefix ur_rl_sb3
```

如果能查到路径，说明当前终端认识这个包。

## 3.2 查可执行入口

```bash
ros2 pkg executables ur_rl_sb3
```

你应该能看到 `train_sac` 和 `eval_sac`。

## 3.3 查话题

```bash
ros2 topic list
ros2 topic echo /joint_states
ros2 topic echo /forward_position_controller/commands
```

## 3.4 构建与环境加载

```bash
cd /home/bophy/d2lros2/ur5/ur_gz
colcon build --symlink-install
source install/setup.bash
```

## 4. 为什么有时“改了代码却没变化”？

通常 4 个原因：

1. 改的是 `src`，运行的是 `install` 旧文件（没 build）
2. build 了但当前终端没重新 `source`
3. 开了多个终端，有的终端没 source
4. 系统里有同名 apt 包，优先级混乱

排查建议：

```bash
which ros2
ros2 pkg prefix ur_rl_sb3
```

并直接打开 `install/.../launch/...` 查看是否与源码一致。

## 5. TF 是什么（你项目里非常关键）

TF 是 ROS2 的坐标变换树。

你项目中：

- 用 `base_link` 到 `tool0`（或候选末端 frame）的 TF，取末端位置
- 根据末端与目标点距离计算奖励和终止条件

如果 TF 拿不到，训练就无法正确计算误差。
